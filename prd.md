# loan-eye PRD：小红书竞品监控平台

**版本：** v0.1
**日期：** 2026-03-11
**状态：** 待评审

---

## 1. 背景与目标

现金贷市场竞争激烈，放心借、微粒贷、百度有钱花、美团借钱等头部产品持续迭代。通过监控这些竞品在小红书上的内容和用户反馈，可以：

- 及时感知竞品产品能力变化（额度、利率、功能迭代）
- 掌握用户真实口碑，发现竞品的投诉热点和差异化机会
- 追踪行业内容传播趋势，辅助产品和营销决策

**核心目标：** 构建一个自动化的小红书竞品监控 Web 平台，每日抓取、分析、可视化呈现竞品动态。

## 5. 分析框架

### 5.0 三层分析目标

| 优先级 | 目标 | 说明 |
|--------|------|------|
| 1 | **C - 借呗自我体检** | 比用户投诉到客服更早发现借呗自身产品问题 |
| 2 | **A - 竞品被动防御** | 竞品有动作时第一时间感知，不靠人工刷小红书 |
| 3 | **B - 差异化洞察** | 基于 C+A 数据，周维度自动生成竞品弱点和差异化机会结论 |

## 6. 功能模块

### 6.1 数据采集模块

> 借呗关键词与竞品关键词统一纳入采集范围，借呗作为"自有产品"配置，在分析层单独处理。

**触发方式：** 每日 02:00 自动执行

**执行流程：**
1. 遍历所有 `is_active=true` 的关键词
2. 调用 MediaCrawler 搜索小红书，抓取近7天笔记
3. 对每篇笔记抓取前100条评论
4. 去重写入：`note_id` 已存在则仅更新互动数据（赞/评/藏/转）
5. 新笔记和评论写入数据库，触发分析流程

**抓取限制：**
- 单次请求间隔随机 2~5 秒（防封号）
- 单关键词每次最多抓取 50 篇笔记
- 失败自动重试3次，仍失败记录告警日志

### 6.2 分析模块

采集完成后自动触发，四个分析器并行执行：

#### 6.2.1 情感分析器（支撑 C）
- 对所有新评论跑 `roberta-base-finetuned-jd-binary-chinese` 模型
- 输出 `positive` / `negative` / `neutral` 标签 + 置信度
- 每日聚合各竞品正面率、负面率
- 用 TF-IDF 提取负面评论的投诉热词 Top10、正面评论的好评热词 Top10
- **借呗专项**：负面评论按以下 6 类打标，每日计数：

| 类别 | 关键词示例 |
|------|-----------|
| 额度问题 | 降额、无故缩减、额度消失 |
| 放款失败 | 提现失败、到账慢、秒拒 |
| 利率感知 | 太贵、比XX高、年化多少 |
| 还款体验 | 扣款失败、提前还款手续费 |
| 客服体验 | 投诉无门、机器人、不解决 |
| 风控误伤 | 征信正常被拒、白名单、申诉 |

- **异常告警**：借呗负面率单日较过去7天均值上升 >15% 时触发告警推送

#### 6.2.2 高传播负面内容识别器（支撑 C）
- 针对借呗，筛选情感标签为 `negative` 且互动量 Top10 的笔记
- 每日输出列表：标题、互动数、发布时间、笔记链接
- 爆款负面笔记（单篇互动量 > 均值3倍）额外标记 `is_hot_negative=true`

#### 6.2.3 产品能力提取器（支撑 A）
- 对笔记正文用规则 + 正则提取结构化产品信息：
  - **额度：** 匹配 `最高X万`、`额度X万`、`借X元`
  - **利率：** 匹配 `日利率X%`、`年化X%`、`月利率X%`
  - **门槛：** 匹配 `征信`、`社保`、`公积金`、`无需XX`、`白户可借`
  - **特色功能：** 匹配 `随借随还`、`秒到账`、`免息X天`、`自动续期`
  - **营销力度：** 匹配 `免息券`、`返现`、`邀请有礼`、`优惠码`；识别高粉账号发布（疑似KOL投放）
- 提取结果写入 `analysis_results.product_features`
- **变化检测**：各维度数值较上月出现显著变化时标记，纳入竞品告警

#### 6.2.4 传播趋势分析器（支撑 A）
- 按竞品统计每日笔记数和总互动量，写入时序数据
- **声量突增告警**：单竞品单日笔记数 > 过去30天均值2倍时触发，自动结合情感判断是正面爆发（大促/新功能）还是负面爆发（投诉潮）
- 爆款识别：单篇互动量 > 该竞品近30天均值3倍，标记 `is_hot=true`
- 行业词维度：统计行业关键词每日声量，识别新兴话题

#### 6.2.5 差异化洞察生成器（支撑 B，周维度）
- 每周一自动汇总上周数据，生成洞察报告：
  - **竞品弱点摘要**：各竞品投诉热词排名，与借呗同类投诉对比，标注借呗相对优势领域
  - **差异化机会点**：竞品高频投诉但借呗无同类问题的领域
  - **市场空白感知**：所有产品均被高频投诉的需求词（行业性痛点）

### 6.3 Web Dashboard

#### 页面零：借呗自检（新增，首页）

**目的：** 借呗产品负责人每日打开第一眼，快速判断有无需要处理的问题

**展示内容：**
- 告警横幅：当天有触发告警时置顶显示（负面率异动 / 爆款负面笔记）
- 投诉分类趋势：6类投诉近30天每日计数折线图
- 今日高传播负面笔记：互动量 Top10 负面笔记列表，含标题、互动数、链接
- 情感趋势：借呗正/负面率近30天走势

#### 页面一：竞品总览

**目的：** 一屏看清各竞品核心指标对比

**展示内容：**
- 顶部：时间范围选择器（近7天 / 近30天 / 自定义）
- 核心指标卡片：各竞品笔记总数、总互动量、正面率
- 竞品能力雷达图：对比额度、利率竞争力、用户口碑、传播声量 4个维度
- 声量趋势折线图：各竞品每日笔记数趋势对比（含声量突增标记）
- 互动量排行榜：各竞品总互动量柱状图

#### 页面二：竞品详情

**目的：** 深入分析单一竞品

**展示内容：**
- 顶部：竞品切换 Tab（放心借 / 微粒贷 / 有钱花 / 美团借钱）
- 产品能力卡片：从笔记中提取的最新 额度/利率/门槛/特色功能/营销活动，变化项高亮
- 情感趋势图：正面率 / 负面率随时间变化折线图
- 词云：投诉热词 + 好评热词并排展示
- 爆款笔记列表：互动量 Top10 笔记，含标题、互动数、发布时间、链接

#### 页面三：行业声量

**目的：** 从行业视角感知赛道整体热度

**展示内容：**
- 行业声量趋势：各行业关键词每日笔记数堆叠面积图
- 热门话题：近期互动量最高的行业笔记 Top20
- 竞品声量占比：各竞品在行业总声量中的饼图占比

#### 页面四：差异化洞察（周报）

**目的：** 每周自动生成竞品弱点和借呗差异化机会的结论性报告

**展示内容：**
- 本周竞品弱点摘要：各竞品 Top3 投诉类别 vs 借呗同类投诉对比
- 差异化机会点：竞品高频投诉但借呗无同类问题的领域
- 市场空白：行业性痛点（所有产品均高频被投诉的需求）
- 历史周报归档列表

## 7. 非功能需求

| 项目 | 要求 |
|------|------|
| 数据延迟 | 每天早上 06:00 前完成前一天数据的采集和分析 |
| 数据保留 | 笔记和评论原始数据保留 180 天，分析结果永久保留 |
| 可用性 | Dashboard 本地部署，无高可用要求 |
| 扩展性 | 竞品关键词通过数据库配置，无需改代码即可新增 |
